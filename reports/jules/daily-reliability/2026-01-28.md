# Daily Reliability Report - 2026-01-28

## Objetivo
Mejorar la observabilidad del World Loop y World Trigger Service para detectar cuellos de botella y asegurar la consistencia del tick rate.

## Cambios Realizados
1.  **Refactor de WorldScheduler**:
    *   Soporte para tareas con nombre (`Schedule(..., name: "TaskName")`).
    *   Detección y logging de "Slow Tasks" (> 10ms).
    *   Detección y logging de "Slow Ticks" (> 50ms).
2.  **Métricas Granulares (ServerMetrics)**:
    *   `RecordSlowWorldTask`: Conteo de tareas lentas.
    *   `RecordSlowWorldTick`: Conteo de ticks que exceden el presupuesto.
    *   `RecordTriggerExecuted`: Conteo de triggers activados.
3.  **Integración en WorldTriggerService**:
    *   Ahora registra métricas cuando un trigger se activa (Enter/Interact).

## Evidencia
