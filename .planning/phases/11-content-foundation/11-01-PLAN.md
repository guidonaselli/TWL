# 11-01-PLAN: Tier 1-4 Items Foundation

## A) Goal
Create `Content/Data/items.json` and populate it with Tier 1-4 consumables, crafting materials, and quest items (Lv1-45), establishing the initial content foundation for The Wonderland Legacy.

## B) Context / Traceability
- **Requirement**: CONT-01
- **Why**: The economy, quest, and crafting systems require a complete, balanced item database. This plan begins fulfilling the WLO-inspired 8-tier progression system by covering the first half (Lv1-45).
- **Dependencies**: None. This is a foundational content task.

## C) Step-by-Step Implementation Instructions
1. **Create the File**: Create `Content/Data/items.json`.
2. **Implement Consumables**:
   - Add HP and SP recovery items for Tiers 1-4 (e.g., Apple, Spring Water, Healing Potion, Mana Potion).
   - Ensure `LevelRequired` correctly scales (Tier 1: Lv1, Tier 2: Lv10, Tier 3: Lv25, Tier 4: Lv45).
   - Ensure prices scale appropriately and `Tradeable` is `true`.
3. **Implement Crafting Materials**:
   - Add basic ores (Copper, Iron, Silver, Gold) and woods (Wood Log, Oak, Rosewood, Spirit Wood) for Tiers 1-4.
   - Add Element Crystals (Earth, Water, Fire, Wind) at Tier 1 to support basic alchemy/crafting.
4. **Implement Legacy Quest Items**:
   - Include specific legacy items verified to be broken in early quest chains:
     - ID 6312: Old Map (Quest 1015)
     - ID 7373: Glowing Mushroom (Quest 9004)
     - ID 7316: Ruins Key (Quests 1020/1021)
     - ID 4739: Strange Relic (Quests 2003/1051/1100/5002)
   - Ensure all quest items have `Tradeable` set to `false`.
5. **ID Rules**:
   - All Item IDs must fall strictly within the 1-9999 range.
   - Ensure NO duplicate IDs.

## D) Testing Requirements
- **Validation**: Ensure `items.json` parses as valid JSON.
- **Test Suite**: Run `dotnet test TWL.Tests/TWL.Tests.csproj --filter "ContentValidationTests"` and verify that item parsing does not throw exceptions and that duplicate ID checks pass.
