# 11-05-PLAN: Pet Roster Completion

## A) Goal
Expand `Content/Data/pets.json` to 50+ entries, defining skills, evolution chains, capture data, and utility roles.

## B) Context / Traceability
- **Requirement**: CONT-03
- **Why**: Pet mechanics are core to TWL. A complete pet database allows players to pursue capture, bonding, and rebirth mechanics across all 8 regions.
- **Dependencies**: Phase 11-04 (pets often reference monster models/stats).

## C) Step-by-Step Implementation Instructions
1. **Open the File**: Modify `Content/Data/pets.json`.
2. **Implement Capture Pets**:
   - Translate 30+ monsters from `monsters.json` into valid pets.
   - Define capture base rates and required items (e.g., Capture Net).
   - Assign starter combat roles (Tank, DPS, Support).
3. **Implement Evolution Chains**:
   - Define multi-stage pet lines (e.g., Baby Dragon -> Young Dragon -> Elder Dragon).
   - Ensure `EvolutionLevel` and `EvolutionId` properties point to valid pet IDs.
4. **Implement Quest Pets**:
   - Create 10+ unique human or legendary pets gated by quest completion.
   - Set `Tradeable` to `false` and `Capturable` to `false`.
   - Provide signature skills for these pets (e.g., Goddess Skills).
5. **ID Rules**:
   - All Pet IDs must be 1000 or greater.
   - Ensure NO duplicate IDs.

## D) Testing Requirements
- **Validation**: Ensure `pets.json` is valid JSON.
- **Test Suite**: Run `ContentValidationTests` to verify no duplicate IDs and that referenced Evolution IDs exist.
