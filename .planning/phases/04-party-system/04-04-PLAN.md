# 04-04-PLAN: Tactical 3x4 Formation and Combat Row Integration

## Goal Description
Implement a 3x4 tactical grid formation for Party members. Tienes toda la razón en que 3x3 quedaba demasiado apretado (9 casilleros para 8 entidades). Al pasar a una grilla de **3 filas por 4 columnas (12 casilleros totales)**, logramos un balance mucho mejor:
- **4 Columnas**: Representan 1 columna "default" designada para cada jugador del party y su pet.
- **3 Filas (Front, Mid, Back)**: Permiten ajustar la profundidad (Row) dentro de su respectiva columna.

Con 8 entidades ocupando 12 casilleros, quedan 4 espacios libres. Esto permite maniobras tácticas reales, armar formaciones asimétricas (ej. acumular fuerzas en un flanco si el enemigo tiene ataques en columna) y da más sentido visual al equipo.
1. **Modificadores de Daño**: Afecta cuánto daño cuerpo a cuerpo (melee) se recibe y se inflige.
2. **Generación de Aggro / Targeteo**: Los enemigos tendrán mayor predisposición a golpear a la fila frontal.
3. **Skills con Área de Efecto (AoE)**: Sentará las bases para futuras skills que ataquen en formas específicas (ej. "Toda una columna", "Forma de cruz", "Splash a casilleros adyacentes").

This fullfills requirement PTY-08.

### Server Architecture (Party Manager)

#### [MODIFY] `TWL.Server\Simulation\Managers\IPartyService.cs`
- Add `(bool Success, string Message) UpdateMemberPosition(int partyId, int characterId, int x, int y)`
- Add `TacticalFormation Formation` to the internal `Party` class.

#### [MODIFY] `TWL.Server\Simulation\Managers\PartyManager.cs`
- Implement `UpdateMemberPosition` with bounding validations (X must be 0-2 (Rows), Y must be 0-3 (Columns)).
- Validate that the target position is not already occupied by another party member.
- When creating a party or adding a new member, assign default grid positions:
  - Player 1 (Leader): Col 0, Mid Row (1, 0). Pet 1: Col 0, Front Row (0, 0).
  - Player 2: Col 1, Mid Row (1, 1). Pet 2: Col 1, Front Row (0, 1)...
- Broadcast a `PartyStateUpdate` whenever the formation changes to sync clients.

#### [NEW] `TWL.Server\Simulation\Networking\Handlers\PartyUpdateFormationHandler.cs`
- Handle incoming `PartyUpdateFormation` packets and route to `PartyManager.UpdateMemberPosition`.

---

### Combat Integration

#### [MODIFY] `TWL.Shared\Domain\Battle\Combatant.cs`
- Add `GridPosition GridPosition { get; set; }`

#### [MODIFY] `TWL.Shared\Domain\Battle\BattleInstance.cs`
- Expose a `GridPosition` translation when passing a Party into a `BattleInstance`. Se asignarán las posiciones de los Jugadores y sus respectivas Pets en la grilla.
- Inside `ApplyDataDrivenSkill` and `ResolveAction` (specifically target selection and `CalculatePhysicalDamage`), apply effects based on the character's row (`GridPosition.X`).
  - **Front Row (X=0)**: Deals 100% melee dmg, receives 100% melee dmg, **High aggro weight** (enemigos targetean más acá).
  - **Mid Row (X=1)**: Deals 80% melee dmg, receives 80% melee dmg, **Medium aggro weight**.
  - **Back Row (X=2)**: Deals 50% melee dmg, receives 50% melee dmg, **Low aggro weight** (ideal para magos/healers).
- *Note: Magic and Ranged damage is unaffected by rows.*
- *Future proofing*: Al tener las entidades en posiciones `(X, Y)` de `0` a `2`, las skills futuras podrán fácilmente filtrar targets (e.g. `targets.Where(t => t.GridPosition.X == targetX)` para atacar toda un columna).

#### [MODIFY] `TWL.Server\Simulation\Managers\CombatManager.cs`
- When calling `StartEncounter`, pull the party's `TacticalFormation` and transpose it onto the generated `ServerCombatant` objects.

   - Boundary checks (X > 2 or Y > 3 is rejected).
   - Collision checks (Moving to an occupied tile is rejected).
   - Default position assignment when members join.
2. **Combat Integration Tests**: Update/Add `BattleInstanceTests.cs` to verify:
   - A combatant in the Back Row receives less melee damage compared to the Front Row given identical defense stats.
   - Magic damage remains identical regardless of the Row.

The verifications will be executed via `dotnet test TWL.Tests/TWL.Tests.csproj`.
