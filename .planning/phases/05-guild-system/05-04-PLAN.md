# Plan 05-04: Guild Shared Storage and Audit Logging

## Goal
Implement and verify the guild shared storage system (GLD-06, GLD-07, GLD-08), including permission-based access, tenure-gated withdrawals, and append-only audit logging.

## Context
- `GuildStorageService` already has basic logic for deposit/withdraw with permissions and tenure.
- `GuildAuditLogService` is implemented for append-only logs.
- `Guild` domain model includes `StorageItems` and `MemberJoinDates`.

## Proposed Changes

### 1. TWL.Server/Simulation/Managers/GuildStorageService.cs
- Refine `DepositItem` and `WithdrawItem` to be more robust.
- Ensure withdrawal tenure is strictly enforced (default 14 days).
- Ensure audit logs are recorded for all withdrawal attempts (success and failure).

### 2. TWL.Tests/Guild/GuildStorageTests.cs (New)
- **TestDeposit**: Verify items can be deposited and guild storage reflects the change.
- **TestWithdrawSuccess**: Verify authorized members can withdraw items.
- **TestWithdrawPermissionDenied**: Verify unauthorized members (e.g., Recruit without permission) cannot withdraw.
- **TestWithdrawTenureGated**: Verify new members cannot withdraw until tenure is met.
- **TestWithdrawInsufficientQuantity**: Verify withdrawal fails if storage doesn't have enough items.
- **TestIdempotency**: Verify duplicate operations (same OperationId) return the same result without double-mutating.
- **TestAuditLogs**: Verify all withdrawal actions are correctly logged in `GuildAuditLogService`.

## Success Criteria
1. Members can deposit items into guild storage.
2. Authorized members can withdraw items if they meet the tenure gate (14 days).
3. Unauthorized members or those under the tenure gate are rejected.
4. All withdrawal attempts are logged in the audit log.
5. Idempotency prevents duplicate storage mutations.
6. All new tests in `GuildStorageTests.cs` pass.
