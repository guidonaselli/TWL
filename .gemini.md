# The Wonderland Legacy - Project Mandates

## Rules of Engagement

1.  **GSD 2 Sync-Check:** Every session must begin by reading `.gsd/STATE.md` and `.gsd/milestones/M001/M001-ROADMAP.md` to identify the current active slice and its next pending task.
2.  **Slice-Level Execution:** When performing work on a Slice (e.g., S08), complete all `- [ ]` tasks in the slice's plan file before moving on. "DONE" means the entire slice is `[x]` in the roadmap.
3.  **No Partial Exits:** Do not stop after a single task if more tasks remain in the current slice, unless blocked by a critical gap or error.
4.  **State Persistence:** Always update `.gsd/STATE.md` and mark tasks `[x]` in the slice plan file before concluding any work session. If a slice is fully complete, mark it `[x]` in `.gsd/milestones/M001/M001-ROADMAP.md`.

## GSD 2 Directory Structure

```
.gsd/
  STATE.md              ← Current position (active milestone, slice, task)
  PROJECT.md            ← Living project description
  DECISIONS.md          ← Append-only architectural decisions
  REQUIREMENTS.md       ← All requirements with status
  milestones/
    M001/
      M001-ROADMAP.md   ← Slice checklist (the master progress tracker)
      M001-CONTEXT.md   ← Milestone context
      slices/
        S08/            ← Each slice has its own directory
          S08-PLAN.md   ← Task checklist for this slice
          tasks/
            T05-PLAN.md ← Individual task implementation plan
            T05-SUMMARY.md ← Post-completion summary
```

## Shell Commands Convention

**Never construct inline shell pipelines** like `dotnet test ... 2>&1 | tail -30`.

Always use the stable PowerShell scripts in `scripts\` via:

```
pwsh -File scripts\<script>.ps1 [args]
```

| Script | Purpose | Example |
|--------|---------|---------|
| `scripts\verify.ps1` | restore → build → test | `pwsh -File scripts\verify.ps1` |
| `scripts\build.ps1` | restore + build only | `pwsh -File scripts\build.ps1 -Config Debug` |
| `scripts\test-filter.ps1` | filtered xUnit test run | `pwsh -File scripts\test-filter.ps1 -Names CharacterRebirthTransactionTests -NoBuild -Tail 30` |
| `scripts\read-log.ps1` | read any file (log/txt/md/trx/json) | `pwsh -File scripts\read-log.ps1 -Path test.log -Tail 50` |
| `scripts\read-runner-info.ps1` | smart lookup of test results and logs | `pwsh -File scripts\read-runner-info.ps1 -Type trx` |

## Branch Protocol

- Code executor branches: `jules/code-M001-SXX-TYY`
- Content executor branches: `jules/content-M001-SXX-TYY`
- Before starting work, check if the branch exists: `git ls-remote --heads origin refs/heads/jules/<branch-name>`
- If it exists, the task is LOCKED — skip to the next `- [ ]` task.

## Agent Identity

-   The agent is proactive, autonomous, and focused on macro-progress.
-   Follow engineering rules in `AGENTS.md`: server-authoritative, inject dependencies, no silent failures.
